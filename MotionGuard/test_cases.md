# MotionGuard Test Cases (Ethernet LAN Cameras)

This document captures validated test modes and operator steps for:
- `Add Camera` with manual RTSP URL
- `Scan ONVIF` for Dahua IP cameras

Working RTSP pattern used for validation:
`rtsp://admin:Tlgcctv@786@192.168.1.121:554/cam/realmonitor?channel=1&subtype=0`

Note: In URLs generated by code, special characters in password are URL-encoded (`@` -> `%40`), which is equivalent.

## Preconditions

1. Camera and PC are on same Ethernet LAN (via switch).
2. Camera RTSP port `554` is reachable.
3. MotionGuard starts successfully:
   `python app\main.py`
4. ONVIF dependencies installed if using scan mode:
   - `onvif-zeep`
   - `WSDiscovery`

## Tested Mode 1: Add Camera (Manual RTSP)

**Status:** Tested

### Steps

1. Open MotionGuard.
2. Click `Add Camera`.
3. Go to `Manual RTSP` tab.
4. Enter:
   - `Camera Name`: e.g., `Gate Cam`
   - `RTSP URL`: full camera URL
   - `Username` / `Password`: optional if already inside URL
5. Click `Test Connection`.
6. Confirm success message (`Connected <width>x<height> @ <fps>`).
7. Click `Save Camera`.
8. In left `Devices` panel, double-click the saved camera to start stream + detection.

### Expected Result

- Camera appears under `Cameras` in device tree.
- Source tile shows live video and status `CONNECTED`.
- Motion state toggles between `NO MOTION` and `MOTION` when movement occurs.
- Motion events appear in right `Motion Events` panel.

## Tested Mode 2: Scan ONVIF (Dahua Cameras)

**Status:** Tested

### Steps

1. Open MotionGuard.
2. Click `Scan ONVIF` (or `Add Camera` -> `ONVIF Discovery` tab).
3. Click `Scan Network (ONVIF)`.
4. Select discovered Dahua device row.
5. Enter ONVIF credentials:
   - Username: `admin`
   - Password: camera password
6. Click `Resolve RTSP Stream`.
7. App switches to Manual tab with resolved URL prefilled.
8. Click `Test Connection`.
9. Enter camera name and click `Save Camera`.
10. Start source by double-clicking camera in `Devices`.

### Expected Result

- Device discovered and stream URL resolved.
- `Test Connection` succeeds after credentials are applied.
- Saved camera starts with live feed and detection.

## Multi-Camera Output Behavior

MotionGuard allows up to **3 active sources** at once.

### Layout Rules

1. **1 active source**: single full tile in center area.
2. **2 active sources**: side-by-side (or stacked if window is portrait).
3. **3 active sources**: 2x2 grid with one empty placeholder cell.
4. **4th source start attempt**: blocked with "Source Limit Reached".

### Operator Steps

1. Add multiple cameras (manual or ONVIF).
2. In `Devices`, double-click each source to start.
3. Confirm `Active: N / 3` in status bar.

## How To View One Camera in Full / Zoomed View

There is no per-tile digital zoom control in current build. Use display modes:

### Option A: Single-source focus

1. Stop other active sources.
2. Keep only one source running.
3. Grid auto-expands that source to full center area.

### Option B: Expand View

1. Click `Expand View` on toolbar.
2. Left and right panels hide; center video area gets more space.

### Option C: Full Screen

1. Click `Full Screen` on toolbar (or press `F11`).
2. Press `Esc` to exit full screen.

## Quick Troubleshooting

1. If VLC works but test fails in ONVIF flow:
   - Re-check credentials in ONVIF and Manual fields.
   - Verify resolved URL path includes Dahua endpoint:
     `/cam/realmonitor?channel=1&subtype=0`.
2. If stream opens but no frames:
   - Check camera stream type (`subtype=0` main, `subtype=1` substream).
3. If ONVIF scan returns nothing:
   - Ensure ONVIF is enabled on camera.
   - Ensure multicast/broadcast is allowed on LAN.
   - Confirm `onvif-zeep` and `WSDiscovery` are installed.
